## Glossary

| Term | Meaning |
| --- | --- |
| Artist | A performer whose profile is managed in the system. |
| Child artist | A protected dependent artist profile managed by authorized adults. |
| Adult artist | An artist with direct access to own permitted data. |
| Coach | A staff user responsible for assigned artists and class records. |
| Parent | An adult linked to one or more child artist profiles. |
| Administrator | A user who manages accounts, roles, settings, and oversight tasks. |
| Attendance | A record of presence for a session or class. |
| Performance record | A record of progress, achievement, or assessment. |
| Injury record | A record of injury-related notes and status. |
| Medical record | A protected record containing health-related information. |
| Emergency contact | A person who should be contacted in urgent situations. |
| Audit log | An immutable history of important system actions. |
| Traceability | The ability to map a requirement to tests, risks, and later decisions. |
| RBAC | Role-based access control. |
| SRS | Software Requirements Specification. |

## Priority Notes

- Security requirements are always treated as Must Have.
- Won't Have items are deferred intentionally, not forgotten.
