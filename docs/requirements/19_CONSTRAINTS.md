# Constraints

- The approved Task 01 architecture must not be redesigned during requirements work.
- No implementation code is written in Task 02.
- Requirements must remain measurable and traceable.
- Security and privacy obligations apply to every sensitive workflow.
- The system must support the academic assessment deliverables as well as production
  quality expectations.
- Feature scope must stay within the approved single-academy baseline.

## Constraint Notes

- Constraints are binding for the current phase unless explicitly revised.
